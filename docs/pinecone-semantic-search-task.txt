Pinecone Duplicate Or Similarity Detection Runbook
==================================================

Purpose
-------
Use Pinecone as the vector database for semantic search in the AWS OpenAI RAG
real-time endpoint deployed through CodePipeline. The endpoint also runs a
recommendation systems task by embedding a seed item, seed text, or user
interest profile and returning the closest Pinecone records. For AI agents, the
endpoint can also write task memories into Pinecone and recall them later with
semantic search. The CI/CD guide task for this repository is duplicate or
similarity detection: embed a candidate item with OpenAI, query Pinecone, and
classify returned records as duplicate, similar, or related by score threshold.

Bash:
./infrastructure/create-pinecone-secret.sh "pcsk-your-pinecone-api-key"

arn:aws:secretsmanager:us-west-1:659613508664:secret:awspineconeapikey1-kiudra

Semantic Search Request
-----------------------
Use this request body against the deployed Lambda Function URL:

{
  "mode": "semantic_search",
  "question": "What recent earnings call transcripts mention EGIO?",
  "top_k": 5
}

Recommendation Systems Request
------------------------------
Use this request body to ask Pinecone for similar records:

{
  "mode": "recommendations",
  "seed_text": "Recent earnings call transcripts and market news mentioning edge cloud, content delivery, and EGIO.",
  "interests": [
    "earnings calls",
    "edge cloud",
    "content delivery",
    "EGIO"
  ],
  "top_k": 5
}

You can also use a Pinecone record as the seed when you already know its id:

{
  "mode": "recommendations",
  "seed_id": "record-id-from-news-demo",
  "top_k": 5
}

Duplicate Or Similarity Detection Request
-----------------------------------------
Use this request body to detect whether a candidate item already exists or is
near an existing Pinecone record:

{
  "mode": "duplicate_similarity",
  "candidate_id": "candidate-egio-edge-cloud-news",
  "candidate_text": "Recent market news and earnings call transcripts discussing EGIO, edge cloud services, content delivery, and infrastructure strategy.",
  "top_k": 5,
  "duplicate_threshold": 0.98,
  "similarity_threshold": 0.85
}

The endpoint embeds candidate_text with OpenAI, queries the configured Pinecone
namespace, excludes candidate_id when present, and returns likely_duplicates,
similar_records, and all matches with a classification field.

Pinecone Memory For AI Agents Task
----------------------------------
Use this request body to store a durable agent memory in Pinecone. The endpoint
embeds the memory with OpenAI, upserts it to the agent-memory namespace, and can
immediately recall related memories when query is included.

{
  "mode": "agent_memory",
  "action": "write",
  "agent_id": "research-agent",
  "task_id": "pinecone-memory-ci",
  "memory_type": "episodic",
  "title": "CodePipeline Pinecone Memory Task",
  "content": "The agent deployed an AWS Lambda Function URL through CodePipeline, used OpenAI embeddings, stored task memory in Pinecone, and verified retrieval for future AI agent context.",
  "query": "What did the agent verify during the CodePipeline Pinecone memory task?",
  "top_k": 3
}

To recall memories without writing a new one:

{
  "mode": "agent_memory",
  "action": "search",
  "agent_id": "research-agent",
  "query": "What did the agent verify during deployment?",
  "top_k": 3
}

Expected Response Fields
------------------------
For semantic search, retrieved_context contains ranked Pinecone matches,
including id, title, content, score, and the original Pinecone metadata.

For recommendation requests, recommendations contains ranked similar records,
including id, title, content, score, and metadata. When seed_id is used, the
seed record is excluded from the returned recommendations.

For agent_memory write requests, stored_memory contains the Pinecone memory id,
namespace, and metadata. When query is included, memories contains ranked memory
matches from the agent-memory namespace.

For duplicate_similarity requests, likely_duplicates contains matches at or
above DUPLICATE_SCORE_THRESHOLD, similar_records contains matches at or above
SIMILARITY_SCORE_THRESHOLD, and matches contains the top classified Pinecone
matches.

CodePipeline Behavior
---------------------
The endpoint deployment buildspec runs Pinecone semantic search and
recommendation systems smoke tests after CloudFormation deploys the Lambda
Function URL. It also runs the Pinecone memory for AI agents task. The smoke
tests now also run the Pinecone duplicate or similarity detection task. The
smoke tests succeed when retrieval_source is "pinecone", the relevant result
list is present, and an agent memory id is returned.

Troubleshooting
---------------
If retrieval_source is "keyword", confirm PINECONE_API_KEY_SECRET_ARN is set and
the Lambda role can read the Pinecone secret.

If Pinecone returns no records, confirm the namespace is news and that records
exist in news-demo. Leave PINECONE_UPSERT_ON_QUERY=false when querying the
existing news index so the endpoint does not insert local sample documents.

If dimensions mismatch, confirm the Pinecone index dimension matches
PINECONE_DIMENSION and the OpenAI embedding model dimension. The current project
uses text-embedding-3-small with 1536 dimensions.
