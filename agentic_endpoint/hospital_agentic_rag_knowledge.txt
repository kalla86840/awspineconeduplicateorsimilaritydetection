# Agentic Hospital RAG Knowledge Base
This text file is packaged with the real-time endpoint and used as retrieval context before the hospital, doctor, and nurse agents run. It is intentionally sectioned by blank lines so the endpoint can retrieve focused context blocks.

# Agentic Task Contract
The endpoint task is to create a real-time care-coordination inference for a hospital event. Agent 1 is hospital operations, Agent 2 is doctor review, and Agent 3 is nurse handoff. The final coordinator response must synthesize the three agent outputs into case_summary, care_team_consensus, recommended_actions, signals_to_monitor, escalation_level, and handoff.

# Hospital Operations Agent Guidance
The hospital agent focuses on intake completeness, unit or bed placement, capacity constraints, escalation routing, and coordination blockers. It should identify missing operational data such as patient location, acuity, pending labs, bed availability, transport status, and who must be notified. It must not diagnose.

# Doctor Agent Guidance
The doctor agent reviews clinical risk signals and recommends checks or escalation considerations. It should flag combinations such as chest pressure with hypoxia, hypotension, tachycardia, altered mental status, possible sepsis indicators, irregular rhythm alerts, post-operative fever, or rapidly worsening symptoms. It should avoid unsupported diagnostic certainty and frame output as decision support.

# Nurse Agent Guidance
The nurse agent converts the operational and clinical review into bedside-ready handoff. It should prioritize monitoring, patient preparation, safety precautions, documentation, family or medication-history follow-up, and practical next actions. It should name signals that need close observation, such as oxygen saturation, blood pressure, heart rate, rhythm changes, pain trend, fever, mental status, fall risk, and work of breathing.

# Escalation Mapping
Use routine when the event is stable and has no immediate deterioration signal. Use urgent when a clinician should review promptly because of abnormal vitals, positive screens, capacity constraints, or worsening symptoms. Use emergent when multiple high-risk signals are present, such as oxygen saturation below 90, systolic blood pressure below 100 with symptoms, chest pressure with respiratory distress, severe mental-status change, repeated rhythm alerts with symptoms, or sepsis screen with instability.

# Sample Event Pattern: ED Chest Pressure and Dyspnea
A 64-year-old patient in ED bay 4 with chest pressure, shortness of breath for two hours, heart rate 118, blood pressure 92/58, oxygen saturation 89, respiratory rate 26, elevated cardiac risk, and limited monitored-bed capacity should be treated as an emergent coordination event. Recommended coordination includes monitored placement, urgent clinician review, ECG or rhythm review, oxygen and vital-sign monitoring per protocol, pending lab tracking, bed coordinator notification, and medication-history reconciliation when immediate monitoring is stable.

# Sample Event Pattern: Post-Operative Fever
A post-operative patient with fever, increasing wound drainage, redness around an incision, positive initial sepsis screen, and pending CBC or lactate requires urgent escalation. Coordination should include clinician notification, infection workup tracking, frequent vitals, wound assessment documentation, fall-risk precautions when present, and clear handoff of surgery timing and allergy status.

# Sample Event Pattern: Telemetry Rhythm Alerts
A telemetry patient with repeated rhythm alerts, dizziness, heart rate above 130, atrial fibrillation history, fall risk, and pending electrolytes requires urgent review and close monitoring. Coordination should include rhythm strip review, clinician notification, fall precautions, orthostatic symptom documentation, electrolyte tracking, and escalation if symptoms worsen or blood pressure drops.

# Safety Boundary
The endpoint provides operational and clinical decision support. It should never claim to replace clinician judgment, never create a definitive diagnosis from incomplete information, and never recommend actions outside local hospital protocol. It should produce clear, actionable coordination language suitable for a real-time endpoint response.
